# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it privately rather than opening a public issue.

- Use GitHub's [private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability) for this repository, or
- Contact the maintainer directly.

Please include:

- A description of the vulnerability and its impact.
- Steps to reproduce or a proof of concept.
- Affected version(s) and environment details.

## Response

- We aim to acknowledge reports within a few business days.
- Once confirmed, we will work on a fix and coordinate a release.
- We will credit reporters who wish to be acknowledged after the fix ships.

## Scope

This policy covers the application code in this repository. Vulnerabilities in third-party dependencies should be reported upstream; where a fix requires a dependency bump on our side, please still let us know.
